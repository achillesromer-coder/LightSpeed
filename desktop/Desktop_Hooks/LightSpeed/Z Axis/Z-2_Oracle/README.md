# Oracle Floor

**Z-Level:** -2
**Version:** 5.1.2
**Status:** Active smart floor

Oracle owns LightSpeed's catalog, proofed knowns, empirical data layer, library,
encyclopedia candidates, datatables, and mounted-source provenance. The active UI
coordinator is the root floor module `Z Axis/Oracle.py`; this folder holds the
owned components and knowledge/data stores.

## Canonical Ownership

- `data/knowns/`: proofed definitions, knowns registry, proofing queue, promotion ledger, and declassification audit.
- `data/library/`: compact library entries, empirical catalog, and promoted knowns.
- `data/encyclopedia/`: encyclopedia candidates and curated reference material.
- `data/datatables/`: curated tables, validation reports, query templates, and analytic overlays.
- `data/catalog/`: entity registry, dataset catalog shell, and promotion queue objects.
- `data/legacy/`: absorbed archive evidence retained for provenance rather than primary UI browsing.

## Current Runtime Flow

Mounted sources are reduced into compact Oracle artifacts first. Morpheus reviews
and edits proofing outputs. Neo consumes proofed knowns and empirical anchors for
bounded proposals. TheConstruct uses simulation-ready anchors. Architect governs
publish packages, while Merovingian records audit and activity evidence.

## Reduction Policy

Do not reintroduce root AI-log, raw-library, or generated-output folders. Raw
evidence stays mounted or under Oracle legacy storage, while user-facing surfaces
must prefer curated catalog, knowns, datatable, and provenance views.
